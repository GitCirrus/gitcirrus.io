Frequently Asked Questions

Who is behind this?

Hi, I'm Joe Cotton. I've been working with the web for nearly 10 years now, in all areas from design to development to systems administration. I started GitCirrus in January 2017 to make my life easier.

How does this make my life easier?

Three reasons:

  1. It's super fast and simple to set up a new website. Just add your domain, give GitCirrus read access to the repository and we'll handle the rest.
  2. There's no management overhead required from you. We take care of everything from security patches to server updates.
  3. It has a powerful, feature-rich control panel that gives you the ability to customise each website deployment and also view traffic stats, configure email forwarding, stream the error logs and much, much more.

How much does it cost?

You get one free website limited to to a maximum 50mb storage and 100mb bandwidth per month.

Upgrading to a Pro account for $7/month allows you to create an unlimited number of websites, however you are limited to one "data pack" which is a storage limit of 50gb and 250gb bandwidth to use each month across all of your websites.

Each additional data pack that you purchase for $7 per month will give you an extra 50gb storage and 250gb bandwidth to use in that month.

For example, paying $28/month will give you 200gb storage and 1000gb of bandwith to use each month. In reality, you are unlikely to ever need to pay more than $7 per month.

What languages/technologies do you support?

Right now we only support PHP versions 5.3, 5.6, 7.0 and 7.1 on Apache 2.4.

In the near future we will be adding support for Hack, Ruby, NodeJS and Python.

You can view all planned and upcoming features on the GitCirrus Roadmap.

Do you provide databases?

Not at the moment, but we plan to host MySQL in the near future. In the meantime you'll need to use a 3rd party database hosting solution such as AWS RDS or Compose MySQL (or even just a Digital Ocean VM running MySQL).

You can view all planned and upcoming features on the GitCirrus Roadmap.

Do you have an API?

Not yet, but we're planning to launch one soon.

You can view all planned and upcoming features on the GitCirrus Roadmap.

Can I use a CDN with this?

Yes! In fact, we recommend that you do use one as it will significantly reduce your monthly bandwith usage. Cloudflare has an excellent (free) CDN that we have been using for years and can strongly recommend.

Tell me about the infrastructure

That's not technically a question, but here goes...

GitCirrus is powered by Docker. Each website that you create is launched in a separate container that gets deployed onto the "GitCirrus Grid"—a fault-tolerant Docker Swarm that runs on a cluster of networked virtual machines hosted by Digital Ocean. The infrastructure is largely autonomous and we make use of the Digital Ocean API to automatically spin up and destroy nodes as the overal cluster load varies without human intervention.

You can view the GitCirrus Gird status page to see what the current situation is—in real time.
